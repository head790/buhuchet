# buhuchet

**AI Native ассистент для индивидуального предпринимателя в РФ.**

Не SaaS. Не приложение. 16 skills для Claude Code, которые превращают его в карманного бухгалтера: счета, акты, договоры, расчёт налогов, взносов, дедлайны.

Работает в **Claude Code CLI**, **Claude Code Desktop** (Windows/Mac) и **Claude Code в IDE** (VS Code, JetBrains).

## Две группы skills

**Skills для государства** (11 шт.) — автоматизируют отношения с налоговой: считают налоги, генерируют счета, акты, договоры, готовят декларацию, напоминают о дедлайнах. Экономят время.

**Skills для себя** (5 шт.) — дают финансовую ясность: сколько реально можно потратить, какая настоящая маржа, по какой цене продавать, хватит ли денег через месяц, насколько тяжела нагрузка на самом деле. Экономят деньги и нервы.

## Поддерживаемые режимы

- **УСН 6%** (доходы)
- **УСН 15%** (доходы минус расходы)
- **Патент (ПСН)**
- **НПД** (самозанятость)

## Быстрый старт

```bash
git clone https://github.com/head790/buhuchet.git
cd buhuchet

# Заполни свои реквизиты
# (замени образцы в data/ своими данными)
nano data/profile.yaml
nano data/clients.yaml

# Очисти примеры (оставь заголовок в CSV)
echo "date,amount,type,description,client,document_number,category" > data/journal.csv
echo "платежи: []" > data/payments.yaml

# Запусти Claude Code (CLI, Desktop или IDE)
claude
```

## Архитектура: AI Native

Этот проект построен по принципу **AI Native** — ИИ не надстройка над существующим ПО, а основа архитектуры. Здесь нет бэкенда, фронтенда, базы данных или сервера. Есть три слоя:

```
              CLAUDE CODE (AI)
     /ip-nalog  /ip-schet  /ip-dedl.
                   |
                   v
            SOURCE OF TRUTH
     profile.yaml  journal.csv  clients.yaml
                   |
                   v
          MCP SERVERS (optional)
        Bank API     FNS     CRM
```

### Три принципа AI Native

**1. Skills — навыки, а не код**
Каждая команда (`/ip-налог`, `/ip-счёт`) — это markdown-файл с инструкциями на естественном языке. Не программа, не скрипт — промпт, который AI интерпретирует и выполняет. Чтобы добавить новую функцию, вы пишете текст, а не код.

**2. Source of Truth — данные как файлы**
Вместо базы данных — `profile.yaml`, `journal.csv`, `clients.yaml`. Человекочитаемые, версионируемые через git, редактируемые в любом текстовом редакторе. AI читает и пишет в них напрямую. Нет ORM, нет миграций, нет схем — файл и есть схема.

**3. MCP — мост к внешнему миру**
Model Context Protocol позволяет AI подключаться к внешним системам: API банка, ФНС, CRM, мессенджеры. В базовой версии проект работает автономно с локальными файлами. С MCP-серверами — становится полноценным агентом, который сам получает выписки, отправляет документы и подаёт отчётность.

### Почему не SaaS?

| | Традиционный SaaS | AI Native |
|---|---|---|
| Где данные | Облако провайдера | У вас, в файлах |
| Как расширять | Ждать фичу от разработчика | Написать новый skill (текст) |
| Интеграции | Только предусмотренные | Любые через MCP |
| Стоимость | Подписка/месяц | Стоимость API-запросов |
| Кастомизация | Настройки в UI | Полная — это ваш код |
| Lock-in | Данные у провайдера | Нет — YAML/CSV ваши |

## Файлы данных

### `data/profile.yaml`

Реквизиты ИП: ФИО, ИНН, ОГРНИП, банк, налоговый режим, ОКТМО. Заполняется один раз.

### `data/journal.csv`

Журнал операций. Формат:

```csv
date,amount,type,description,client,document_number,category
2025-01-15,150000.00,income,Разработка сайта,ООО Ромашка,1,услуги
2025-01-20,5000.00,expense,Хостинг VPS,,,хостинг
```

Добавляется автоматически через `/ip-доход` и `/ip-расход`.

### `data/clients.yaml`

Реквизиты контрагентов: название, ИНН, КПП, банк, тип (юрлицо/ип/физлицо).

### `data/payments.yaml`

Уплаченные налоги и взносы. Добавляйте вручную после каждой оплаты.

## Отказ от ответственности

Это **не** сертифицированное бухгалтерское программное обеспечение. Проект автоматизирует рутинные расчёты и генерацию документов, но не заменяет профессиональную бухгалтерскую или налоговую консультацию.
