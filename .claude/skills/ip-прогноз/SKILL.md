Покажи прогноз денежного потока на 3 месяца.

Аргументы: $ARGUMENTS (не используются)

## Инструкции

1. Прочитай `data/profile.yaml` — налоговый режим
2. Прочитай `data/journal.csv` — история доходов и расходов (для расчёта средних)
3. Прочитай `data/payments.yaml` — уплаченные платежи (чтобы понять, что ещё предстоит)

## Расчёт

### Прогноз доходов

На основе последних 3-6 месяцев рассчитай средний месячный доход. Если данных мало — используй все доступные.

```
средний_доход = сумма_доходов_за_N_месяцев / N
```

### Прогноз расходов

```
средние_расходы = сумма_расходов_за_N_месяцев / N
```

### Известные платежи

На основе налогового календаря (как в `/ip-дедлайны`) определи фиксированные платежи на ближайшие 3 месяца:
- Авансовые платежи УСН (28-е число: апрель, июль, октябрь)
- Страховые взносы (рекомендуемый поквартальный платёж)
- 1% свыше 300К (до 1 июля)
- Стоимость патента (если применимо)

### Баланс по месяцам

```
для каждого из 3 месяцев:
    приход = средний_доход
    расход = средние_расходы + налоговые_платежи_в_этом_месяце
    баланс_на_конец = баланс_на_начало + приход - расход
```

## Формат вывода

```
ПРОГНОЗ ДЕНЕЖНОГО ПОТОКА
На основе: средние за последние {N} мес.
Средний доход: {сумма} руб./мес. | Средний расход: {сумма} руб./мес.

             {месяц 1}      {месяц 2}      {месяц 3}
─────────────────────────────────────────────────────────
Приход       {сумма}        {сумма}        {сумма}
Расходы     -{сумма}       -{сумма}       -{сумма}
Налоги      -{сумма}       -{сумма}       -{сумма}
─────────────────────────────────────────────────────────
Итого        {сумма}        {сумма}        {сумма}
Баланс       {сумма}        {сумма}        {сумма}

Налоговые платежи в прогнозе:
  {дата} — {описание} — {сумма} руб.
  {дата} — {описание} — {сумма} руб.
```

Если баланс в каком-то месяце становится отрицательным:
"⚠️ КАССОВЫЙ РАЗРЫВ в {месяце}: дефицит {сумма} руб. Рекомендуется: ускорить оплату от клиентов / отложить крупные расходы / заранее уплатить взносы для вычета из налога."
